remjs

Streaming JavaScript state serialization. Wrap a state tree in a proxy, get a stream of structured ops, rebuild the same state anywhere.

GitHub Open full demo ↗

remjs is a tiny library (~500 LOC) for turning JS state mutations into a transport-agnostic op stream. You wrap a plain object with createStateStream; you mutate it normally; every change gets emitted as a structured op batch. On the receiving side, applyOps rebuilds identical state against a plain object. There's no server, no framework, no schema — you pick the wire.

remjs

Streams JS execution state — objects, arrays, Maps, Sets, nested trees. The source-of-truth layer.

remdom

Streams DOM mutation state — nodes, attributes, text. The rendered-output layer.

Usage

import { createStateStream, applyOps, decode } from 'remjs';

// Sender: wrap a plain state object
const { state, snapshot } = createStateStream(
  { count: 0, todos: [], user: { name: 'Alice' } },
  {
    onOps: (ops) => ws.send(JSON.stringify(ops)),
    batch: 'microtask',
  },
);

// Mutate normally — ops are emitted automatically
state.count++;
state.todos.push('buy milk');
state.user.name = 'Bob';

// Receiver: start from a snapshot, apply incoming batches
let mirror = decode(snapshot().value);
ws.onmessage = (ev) => {
  mirror = applyOps(mirror, JSON.parse(ev.data));
};

What ops look like

{ "type": "set",    "path": ["count"],        "value": 1 }
{ "type": "set",    "path": ["user", "name"], "value": "Bob" }
{ "type": "delete", "path": ["todos", "0"] }
{ "type": "mapSet", "path": ["cache"], "key": "k", "value": 42 }

Plus mapDelete, mapClear, setAdd, setDelete, setClear, and snapshot. Values that JSON can't express — undefined, NaN, Date, BigInt, Map, Set, RegExp — are tagged with a reserved __remjs key at encode time and restored on decode.

Transports

remjs has no built-in transport — onOps hands you a batch and it's your job to ship it. Reasonable options:

  • WebSocket — multi-client sync
  • postMessage — to a Worker or iframe
  • BroadcastChannel — peer-to-peer between tabs, no server
  • WebRTC data channel — peer-to-peer across machines
  • A local applyOps call — in-page mirroring (the shuffleboard above works this way — its entire "transport" is a function call)
  • A log file — for record/replay debugging

Scope

  • Captures mutations on objects, arrays, Maps, Sets, and nested trees.
  • Does not capture closures, local variables, or primitives — use a build-time source transform for those.
  • Applies ops in order. Concurrent multi-writer scenarios need a CRDT/OT layer above remjs.
  • The receiver trusts the op stream; there's no schema enforcement.
