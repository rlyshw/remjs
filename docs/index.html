remjs

Streaming JS state serialization

Wrap a JavaScript state tree in a proxy, get a stream of structured ops describing every mutation, rebuild identical state anywhere. No framework, no transport, ~500 lines of TypeScript.

How it works

createStateStream wraps a plain object with a deep Proxy that intercepts every property write, array method, and Map/Set mutation. Each mutation becomes a structured op. You ship the ops over any transport — WebSocket, postMessage, BroadcastChannel, an in-process function call — and the receiver calls applyOps on a plain object to rebuild the same state.

source side
import { createStateStream } from 'remjs';

const { state, snapshot } = createStateStream(
  { count: 0, user: { name: 'Alice' } },
  { onOps: (ops) => transport.send(ops) },
);

// mutate normally — ops are emitted automatically
state.count++;
state.user.name = 'Bob';
receiver side
import { applyOps, decode } from 'remjs';

let mirror = decode(initialSnapshot.value);
transport.onMessage = (ops) => {
  mirror = applyOps(mirror, ops);
};

The library is transport-agnostic. It gives you an op stream and a receiver; you decide how to move bytes. The demo below does it with a single function call — both sides live in the same browser tab.

What's interceptable

remjs scopes itself to application state: things you keep in plain objects — Redux stores, Zustand stores, React state trees, arbitrary JS object graphs. Closures and primitive locals need a build-time source transform; everything else goes through the Proxy.

Object property writes

Plain set / delete via the generic Proxy trap.

Array mutations

push / pop / splice / sort decompose into per-index sets + length change.

Map & Set mutations

set / delete / clear / add wrapped per method — internal slots work.

Nested trees

Child objects wrapped lazily on read with the extended path.

Special types

Date, RegExp, BigInt, Map, Set, NaN, ±Infinity survive JSON.stringify.

Closure state

Not observable via Proxy — needs a source transform.

Performance

Headless benchmark on Node 24: 250 rigid bodies, 4 field updates per body per frame, full JSON.stringify → JSON.parse → applyOps round-trip. Zero desync across 600 frames.

The protocol

JSON over the wire. Paths are arrays of string keys. Values that JSON can't express — undefined, NaN, Date, BigInt, Map, Set, RegExp — are tagged with a reserved __remjs key at encode time and restored on decode.

example op stream
{ "type": "set",    "path": ["count"],        "value": 1 }
{ "type": "set",    "path": ["user", "name"], "value": "Bob" }
{ "type": "delete", "path": ["todos", "0"] }
{ "type": "mapSet", "path": ["cache"], "key": "k", "value": 42 }
{ "type": "snapshot", "value": { /* full tree */ } }

Object / array

  • snapshot — full tree serialization
  • set — property write at path
  • delete — property delete at path

Map / Set

  • mapSet / mapDelete / mapClear
  • setAdd / setDelete / setClear

Non-goals

remjs deliberately doesn't do these. If you need them, add a layer above:

Transport

No sockets, no auth, no framing. You pick the wire.

Conflict resolution

Ops apply in order. Multi-writer needs a CRDT or OT layer.

Schema enforcement

The receiver trusts the op stream.

Closure capture

Use a source transform for locals and closures.
